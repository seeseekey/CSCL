FTPConnection.StrictReturnCodes Property

Controls whether or not checking of return codes is strict.

public bool StrictReturnCodes {get; set;}

Property Value

true if strict return code checking, false if non-strict.

Remarks

Some servers return non-standard reply-codes. When this property is false only the first digit of the reply-code is checked, thus decreasing the sensitivity of edtFTPj to non-standard reply-codes. The default is true meaning that reply-codes must match exactly.
